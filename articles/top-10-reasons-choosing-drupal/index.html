Top 10 reasons for choosing Drupal

So why should you use Drupal for your software project? Here are the top 10 reasons...

Open Source

Drupal is Open Source software which is free to use, so there are no licence fees to pay

Customisable

There’s a huge library of modules that can be used to add features to Drupal. Or if you have the expertise you can build new modules and contribute them back to the Drupal community (we’ve contributed over 50 so far!)

SEO

Drupal is Search Engine Friendly with auto-generated urls that work for users and search engines. In Drupal 8 there are a host of new features and tools to improve optimization such as OpenGraph and Yoast

Mobile optimized

Drupal 8 is mobile optimized out of the box and there are additional tools such as AMP which are easily implemented with a little tweaking (we can show you how)

Scalable

Drupal is flexible enough to work with large and small projects (and so are we)

Searchable

All content is indexed and searchable, and Drupal can also generate site maps to submit to search engines for better SEO

Content Management System (CMS)

Drupal has a user-friendly editing interface. With minimal training even non-technical staff are able to update the site and create new content

Permission system

Drupal has a built-in permission system where users can have different roles allocated. This makes it easy to differentiate who can view and edit content and site architecture

Drupal Community

There is a whole community of Drupal users who work together to build and review code. There is a security team (we’re part of that!) to keep the code secure and there are forums to get help and support

DrupalCon

A social (and business side) to Drupal. In a different European City each year it’s a great opportunity to catch up on the latest developments, share ideas (and a few drinks!)
